「差分バッチ(JSON)」を渡します。

【差分バッチの仕様】
- 差分バッチは RFC6902 の JSON Patch 規格に準拠します。
- jsonPatch フィールドは JSON Patch オペレーションの配列です。
- 各要素は { "op": "add|remove|replace", "path": "/...", "value": ... } の形式です。
- path は JSON Pointer (RFC6901) に準拠し、例として /version, /stage, /items/0, /items/- などが含まれます。

【あなたのタスク】
- 元JSONに対して jsonPatch を適用し、差分を反映した「最新の完全JSON」を構築してください。
- 出力は Json Schema v1.0.10 に準拠した 1 個の JSON オブジェクトのみとします。
- 手動で変更した値を最優先し、整合性が取れるように関係する他のフィールドを再推論してください。
- nlCollentionに修正指示が入力されている場合は、それに従って構造物ごとに整合性が取れるように再度推論してください。修正JSON出力する際にはnlCollectionに空文字列を入力しておいてください。
- JSON 以外（説明文・マークダウン・コメント）は一切出力しないでください。
